# 優秀海報評選系統 — 後端安裝說明（白話版）

---

## 一、先講「後端」到底是什麼

你現在的評分系統，其實是**兩個東西**在合作：

| | 是什麼 | 放在哪 | 負責什麼 |
|---|---|---|---|
| **前端** | 網頁 | GitHub（`poster-award.html`） | 委員看到的畫面、按鈕、評分表 |
| **後端** | 一段程式 | Google 試算表裡的「Apps Script」 | 把分數**存進試算表**、再**讀出來** |

網頁本身**不會記住任何東西**。委員在手機上按了「提交評分」，那筆分數要有人接住、寫進 Google 試算表——接住的那個人就是「後端」。

你現在初審系統用的後端，網址長這樣（在 `poster-review.html` 裡）：

```
https://script.google.com/macros/s/AKfycbz2njUGr6eORWe6V5iMITA_dbpu.../exec
```

新的優秀海報評選，**用的是同一個後端**。但那段程式目前只認識「初審評分」，不認識「現場評分」。所以要**多教它幾句話**——這就是 `award-gas-snippet.gs` 的用途。

---

## 二、不裝會怎樣？

**委員端照常能用**：可以登入、看初審成績、評分、送出，分數也會存起來。

**但後台看不到即時排名**。你要的第 3 點（現場委員評完，後台馬上看到分數、確認名次）就做不到。

所以——**這個一定要裝**。好消息是只要做一次，5 分鐘。

---

## 三、安裝步驟

### 步驟 1　打開 Apps Script

1. 打開你放**初審評分資料**的那份 Google 試算表
2. 上方選單點 **「擴充功能」** → **「Apps Script」**
3. 會跳出一個寫程式的頁面

> 確認方法：這頁應該看得到你原本的程式碼，裡面找得到 `function doGet` 這一行。找不到就是開錯試算表了。

---

### 步驟 2　貼上新程式

1. 在左邊「檔案」旁邊，點 **「＋」** → 選 **「指令碼」**
2. 檔名打 `award`，按 Enter
3. 右邊會出現一個空白檔案，**把裡面原本的幾行字全部刪掉**
4. 打開 `award-gas-snippet.gs`，**整份複製貼上**
5. 按上方的 **💾 儲存**

---

### 步驟 3　讓原本的程式認識新功能 ⚠️ 最重要的一步

1. 在左邊點回你**原本的程式檔**（通常叫 `程式碼.gs` 或 `Code.gs`）

2. 用 `Ctrl + F` 搜尋 `function doGet`，找到這一行

3. 在它的**正下方**，貼上這兩行：

   ```javascript
   var __aw = handleAwardAction(e);
   if (__aw) return __aw;
   ```

4. 貼完會長這樣：

   ```javascript
   function doGet(e) {
     var __aw = handleAwardAction(e);      // ← 新增這行
     if (__aw) return __aw;                // ← 新增這行

     var action = e.parameter.action;      // ← 原本的程式，不要動
     ...
   }
   ```

5. 按 **💾 儲存**

> **這兩行在做什麼？**
> 就像總機接電話。原本總機只認識「初審」的電話，這兩行是跟它說：「如果來電是找**優秀海報**的，轉給新來的同事處理；不是的話，你照原本的方式接。」
>
> 所以**原本的初審功能完全不受影響**。

---

### 步驟 4　重新部署 ⚠️ 沒做這步等於沒裝

1. 右上角點 **「部署」** → **「管理部署作業」**
2. 找到現有的那筆，點右上角的 **鉛筆 ✏️（編輯）**
3. **「版本」下拉選單 → 選「新版本」**（← **關鍵**，選錯就不會生效）
4. 點 **「部署」**
5. 完成

> **網址不會改變**，所以 `poster-review.html`、`poster-award.html` 都不用改任何東西。

---

## 四、確認裝好了沒

### 方法 A：在 Apps Script 裡測（最快）

1. 回到剛剛貼上程式的 `award` 檔案
2. 上方函式下拉選單選 **`awardSetupCheck`**
3. 點 **「執行」**
4. 第一次執行會跳出授權視窗 → 點「檢閱權限」→ 選你的 Google 帳號 → 「進階」→「前往...(不安全)」→「允許」
5. 下方「執行記錄」出現類似這樣就成功：

   ```
   「優秀海報評分」現有 0 列資料
   「優秀海報名次」現有 0 列資料
   安裝正常，記得在 doGet(e) 開頭加入 handleAwardAction 那兩行並重新部署。
   ```

6. 回到 Google 試算表，最下方分頁會多出兩張表：
   **「優秀海報評分」**、**「優秀海報名次」**

### 方法 B：在網頁上看

用**管理員帳號**登入 `poster-award.html`：

- 看到黃色警告框「⚠ 後端尚未安裝...」→ **還沒裝好**，回頭檢查步驟 3、4
- 沒有黃色警告框，而且右上角出現「⏱ 自動更新」按鈕 → **裝好了** ✅

---

## 五、它會動到我原本的資料嗎？

**不會。** 這段程式只碰它自己新建的兩張表：

- **「優秀海報評分」**——現場委員送出的分數
- **「優秀海報名次」**——你確認的名次

原本的初審資料、委員帳號、指派表，**一律不碰、不讀、不改**。

要移除的話：刪掉 `award` 檔案 + 步驟 3 那兩行 + 重新部署，就恢復原狀。

## 六、完整操作流程

### 【約會前 2 週】秘書處：建立委員帳號

1. 在 `poster-review.html` 管理後台 →「評分結果」
2. 把入選的論文，**展出方式**點成「**優秀海報**」
3. 用**管理員帳號**登入 `poster-award.html` → 上方切到 **「👤 現場委員帳號」**
4. 點 **「＋ 一鍵建立」**（預設 3 組）
   - 系統自動產生 `award01`、`award02`、`award03`
   - 密碼隨機 8 碼，已排除容易看錯的字元（`0` `O` `1` `l` `I`）
   - 姓名先帶「現場評審委員1～3」，可之後再改
5. 點 **「📋 複製全部帳密」** → 直接貼到 Email 寄給 3 位委員

   複製出來的內容長這樣：

   ```
   TSMCSD 2026 優秀海報論文評選系統｜評審委員登入資訊
   登入網址：https://ling3567.github.io/2026annualconference/poster-award.html
   ──────────────────────────────
   現場評審委員1
   　帳號：award01
   　密碼：vVRVaF4c
   ...
   ```

> **帳號寄出 = 開通。** 沒有另外設開放日期，委員收到信就能登入。
>
> 現場委員帳號**不會**被指派任何書面初審工作，他們登入初審系統也看不到待評項目，兩邊不會互相干擾。

### 【會前～當天】3 位現場委員：預覽

委員登入後就能**預覽所有入選海報**：

- 論文題目、作者群、完整摘要（研究動機／方法／結果／結論）
- **論文全文 PDF** —— 可以在頁面內直接預覽，也可以用「↗ 新分頁開啟」
- 書面初審成績：各項平均得分表、總分、初審評審意見

### 【當天】3 位現場委員：評分

1. 手機或平板開 `poster-award.html` 登入
2. 點論文 → 上方顯示該篇的初審成績與全文 PDF
3. 只需評 **「口頭報告之結構性、完整性及海報製作品質」** 一項（滿分 20），並填評審意見
4. 按「提交評分」→ 自動跳下一篇

### 【當天】秘書處看即時排名

1. 用管理員帳號登入 `poster-award.html`
2. 點 **「⏱ 自動更新」**（每 20 秒自動抓新分數，有人送出會跳提示）
3. 畫面即時顯示每篇：

   ```
   初審 87.5 ＋ 現場 17 ＝ 104.5
   3/3 位已評
   ```

4. 三位委員都評完後，點 **「依總分自動帶入名次」**
5. 需要調整就用下拉選單手動改
6. 點 **「儲存名次」** 寫回試算表
7. 點 **「匯出 CSV」** 拿到完整成績表（含每位委員給分與評審意見）

## 七、卡住了怎麼辦

| 狀況 | 原因 | 怎麼解 |
|---|---|---|
| 管理端一直顯示黃色警告框 | 步驟 4 沒選「新版本」 | 重做步驟 4，版本一定要選「新版本」 |
| 委員送得出分數，後台看不到 | 同上 | 同上 |
| `handleAwardAction 未定義` | 步驟 2 沒存檔，或貼到錯的專案 | 確認 `award` 檔案存在且已儲存 |
| 名單是空的 | 還沒設「優秀海報」 | 回 `poster-review.html` 設展出方式 |
| 委員登入失敗 | 帳號被停用或打錯 | 到「👤 現場委員帳號」確認狀態為「啟用中」，密碼可直接在列表看到 |
| PDF 預覽一片空白 | 部分手機瀏覽器不支援內嵌 PDF | 改點「↗ 新分頁開啟」；或確認該 PDF 的 Google Drive 權限為「知道連結的人可檢視」 |
| 某篇沒有 PDF 按鈕 | 該篇投稿時未上傳全文 | 顯示「本篇未提供」，請以摘要與現場海報為準 |
| 一鍵建立的帳號編號重複 | — | 系統會自動從既有最大編號往後接，不會覆蓋 |
| 想清掉測試分數 | — | Apps Script 執行 `awardClearScores`（只清現場評分，初審不動） |

## 八、要改設定的話

都在 `poster-award.html` 上方，`<script>` 開頭附近：

| 要改什麼 | 找這個 | 預設值 |
|---|---|---|
| 現場評分項目、配分 | `AWARD_CRITERIA` | 口頭報告…（20 分） |
| 現場委員人數 | `ONSITE_JUDGE_COUNT` | `3` |
| 現場委員帳號前綴 | `ONSITE_USER_PREFIX` | `award`（`award01`…） |
| 初審滿分 | `REVIEW_MAX` | `100` |
| 要不要給現場委員看到初審委員姓名 | `SHOW_REVIEWER_NAMES` | `false`（顯示「委員1、委員2」） |
| 後台自動更新頻率 | `AUTO_INTERVAL_MS` | `20000`（20 秒） |
